Ontology as Operating Layer

本体论：把行业理解变成可计算结构

它不是一套术语表，而是一种把业务知识组织成对象、关系、动作和计算逻辑的方法论。

01 / 方法论

本体论先回答一个问题：业务世界如何被系统理解？

对业务知识进行结构化建模：把人脑里对行业的理解，变成计算机能读懂、能查询、能计算、能执行的结构。

1

哪些东西存在：企业、材料、设备、订单、人员、流程。

2

它们如何连接：供应、生产、归属、影响、触发。

3

系统如何行动：写回图谱、触发流程、启动分析。

02 / Palantir Foundry

企业软件里的本体层：把数据建模成业务图谱

Palantir 是最早把本体论引入企业软件平台的公司之一。在 Foundry 中，客户企业的设备、订单、人员、流程，会被按照 对象、链接、动作 建模成一张图谱。

关键不是“把数据库换个名字”，而是给数据加上一层业务语义，让上层应用面向业务对象工作。

03 / 语义中台

企业数字资产的语义中台：统一对象、链接、动作、函数。
原始数据
数据库、表、日志、接口、文档、业务系统。
04 / 核心要素

四个基本件，刻画一个行业

O

对象 Object

行业里存在的实体，以及它们的属性。

L

链接 Link

对象之间的关系，说明谁影响谁、谁连接谁。

F

函数 Function

定义在对象和链接之上的计算逻辑。

A

动作 Action

图谱数据的写操作入口，以及后续触发行为。

05 / 新能源行业

对象定义“谁存在”，链接定义“关系是什么”

碳酸锂 原材料对象 Object
正极材料 材料对象，被上游原料影响 Link: 原材料
宁德时代 电池企业对象 Link: 供应
理想汽车 整车企业对象 Link: 供给

例如：“碳酸锂是正极材料的原材料”，这句话在系统里不只是文本，而是一条可遍历、可计算、可被应用调用的链接。

06 / 计算逻辑

函数告诉系统：如何在结构上做分析

价格传导函数：输入任意一个原材料节点和价格变化幅度，系统沿供应链关系向下遍历，计算每一跳的成本影响。

碳酸锂 输入：价格 -10% source
正极材料 计算：原料成本变化 impact 1
电芯 计算：BOM 成本变化 impact 2
整车 计算：毛利与定价空间 impact 3
07 / 闭环

动作把分析结果写回图谱，或触发后续操作

对象

定义谁存在。

链接

定义关系是什么。

函数

定义怎么在结构上计算。

动作

定义怎么写回图谱，或触发下一步。

本体层的价值，是把企业数据从“能存储”提升到“能理解、能计算、能行动”。
